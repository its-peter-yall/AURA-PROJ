**Create AURA-CHAT/server/routers/settings.py:**

Follow the Example 1 pattern from 10-RESEARCH.md. Create a FastAPI `APIRouter` with `prefix='/api/v1/settings'` and `tags=['Settings']`.

**Dependency injection helpers** (module-level functions used with `Depends()`):
- `get_redis()`: Return the app's async Redis client. AURA-CHAT server uses `redis.asyncio` — check how other routers obtain their Redis connection (look at `server/main.py` startup or existing Redis usage). If no shared Redis connection exists in the server layer, create one using `redis.asyncio.from_url(os.getenv('REDIS_URL', 'redis://localhost:6379'))` as a module-level singleton.
- `get_settings_store(redis=Depends(get_redis))`: Return `SettingsStore(redis)`.
- `get_key_manager(redis=Depends(get_redis))`: Return `KeyManager(redis)`.
- `get_model_cache(redis=Depends(get_redis))`: Return `ModelCache(redis, get_default_router())`.

**Pydantic request/response models:**
```python
class DefaultModelUpdate(BaseModel):
    provider: str
    model: str

class ApiKeyCreate(BaseModel):
    api_key: str
```

**Endpoints (8 total):**

1. `GET /defaults` → `get_defaults(store=Depends(get_settings_store))` — returns all use-case defaults
2. `PUT /defaults/{use_case}` → validate use_case in ('chat', 'embeddings', 'entity_extraction'), call `store.set_default()`, return updated value. Reject unknown use_cases with 400.
3. `GET /providers/{provider}/models` → `list_models(provider, refresh: bool = False, cache=Depends(get_model_cache))` — return cached or fresh model list. Query param `?refresh=true` for force refresh.
4. `GET /models` → aggregate models from all providers (call `cache.get_models()` for each registered provider, or use `router.list_models()` directly with caching)
5. `POST /providers/{provider}/api-key` → store encrypted key, optionally validate via health check, return `{provider, masked_key, valid}`.
6. `GET /providers/{provider}/api-key` → return masked key or 404.
7. `DELETE /providers/{provider}/api-key` → delete key or 404.
8. `POST /providers/{provider}/validate` → re-validate stored key via health check.

**Error handling:** Wrap operations in try/except for `ValueError` (missing master key), `ModelRouterError` (provider issues), return appropriate HTTP status codes (400, 404, 500).
