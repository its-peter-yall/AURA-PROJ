```python
This module preserves the legacy import surface so existing consumers
(rag_engine.py, llm_entity_extractor.py, etc.) continue to work without
code changes. All actual LLM work is delegated to the shared model_router.

from typing import Any, AsyncGenerator, Dict, List, Optional

)
```

**Re-exported names the façade MUST provide (consumers depend on these):**

1. `GenerationConfig` class — Create a thin shim:
   ```python
   class GenerationConfig:
       """Thin config shim — stores kwargs for model_router translation."""
       def __init__(self, **kwargs):
           self._kwargs = kwargs
           for k, v in kwargs.items():
               setattr(self, k, v)
       def to_dict(self):
           return dict(self._kwargs)
   ```

2. `VertexAIRequestError` class — Subclass of ModelRouterError for backward compat:
   ```python
   class VertexAIRequestError(ModelRouterError):
       """Legacy error type preserved for existing except clauses."""
       def __init__(self, message='', **kwargs):
           super().__init__(message, provider='vertex_ai', **kwargs)
   ```

3. `get_model(model_name: str)` → Returns `VertexCompatModel(model_name)`. No test mode check needed — VertexCompatModel handles it via the router.

4. `init_vertex_ai(location=None)` → No-op function (router handles initialization):
   ```python
   def init_vertex_ai(location: Optional[str] = None) -> None:

       pass
   ```

5. `normalize_model_name(model_name: str)` → Delegate to model_router:

   ```python
   from model_router.providers.vertex_ai import _normalize_vertex_model_name
   ```

6. `generate_content(model, contents, *, generation_config, safety_settings=None)` → Use the compat model:
   ```python
   def generate_content(model, contents, *, generation_config=None, safety_settings=None):
       if hasattr(model, 'generate_content'):
           return model.generate_content(contents, generation_config=generation_config, safety_settings=safety_settings)
       compat = VertexCompatModel(str(model))
       return compat.generate_content(contents, generation_config=generation_config, safety_settings=safety_settings)
   ```

7. `generate_content_stream(*, model_name, contents, generation_config, system_instruction=None, thinking_config=None)` → Delegate to router.stream():
   ```python
   async def generate_content_stream(*, model_name, contents, generation_config=None,
                                      system_instruction=None, thinking_config=None) -> AsyncGenerator[Dict[str, Any], None]:
       router = get_default_router()
       request_kwargs = {'model': model_name, 'contents': contents}
       if system_instruction:
           request_kwargs['system_instruction'] = system_instruction
       if thinking_config:
           request_kwargs['thinking_config'] = thinking_config
       if generation_config:
           cfg = generation_config.to_dict() if hasattr(generation_config, 'to_dict') else (generation_config if isinstance(generation_config, dict) else {})
           for key in ('temperature', 'max_output_tokens', 'top_p'):
               if key in cfg:
                   request_kwargs[key] = cfg[key]
       async for chunk in router.stream(GenerateRequest(**request_kwargs)):
           yield {'type': chunk.type, 'text': chunk.text}
   ```
   **Note:** Yields dicts (not StreamChunk objects) for backward compatibility — existing consumers use `chunk.get('type')` and `chunk.get('text')`.


**Rewrite the EmbeddingService class** in AURA-CHAT/backend/utils/embeddings.py to delegate actual embedding API calls to `router.embed()` instead of direct REST API calls.

**Current state:** The file (~400 lines) implements embeddings via direct Vertex AI REST API calls using `google.auth` credentials and `requests` library. It has sophisticated batching, rate limiting, and retry logic.

**Migration approach (per research Pitfall 3):** Keep the batching, rate limiting, and retry orchestration logic in EmbeddingService. Only replace the low-level API call with `router.embed()`.

**Specific changes:**

1. **Remove direct SDK/REST imports:** Delete imports of `google.auth`, `google.auth.transport.requests`, `requests`, and any direct `google.cloud.*` imports. Keep standard library imports (os, time, logging, etc.).

2. **Add model_router imports:**
   ```python
   import asyncio
   from model_router import get_default_router
   from model_router.compat import _run_sync
   ```

3. **Replace the `_call_embedding_api()` or `_embed_single_batch()` method** (the method that makes the actual REST/SDK call for one batch of texts) with a router delegation:
   ```python
   def _embed_batch_sync(self, texts: list[str]) -> list[list[float]]:
       """Delegate embedding to model_router with sync bridge."""
       router = get_default_router()
       return _run_sync(router.embed(texts))
   ```

4. **Keep the outer batching loop** that splits large lists into batch_size chunks and calls the inner method for each chunk. Keep the rate limiting logic (RPM tracking, sleep-if-needed). Keep the retry logic with exponential backoff.

5. **Update the retry logic** to catch `ModelRouterError` instead of HTTP status codes / `requests.exceptions`. Map the router's `RateLimitError` to the existing retry-on-429 behavior.

6. **Keep test mode handling:** The router already returns canned embeddings in test mode (768-dim zero vectors). The EmbeddingService's test mode check can be simplified to just delegate to the router (which handles test mode internally). If the current code has a `_test_mode` flag with deterministic fake embeddings, keep it for backward compat OR delegate to router's test mode.

7. **Preserve the public API exactly:**
   - `EmbeddingService()` constructor with same config (batch_size, rpm_limit, etc.)
   - `embed_texts(texts)` or `get_embeddings_batch(texts)` — whatever the current public method names are
   - `embed_single(text)` — single text convenience method
   - Any other public methods used by rag_engine.py

**Do NOT:**
- Remove batching logic (router.embed() handles a single batch, not multi-batch orchestration)
- Remove rate limiting (still needed for high-volume KG processing)
- Change the class name or public method signatures
- Import vertexai, google.auth, or requests
