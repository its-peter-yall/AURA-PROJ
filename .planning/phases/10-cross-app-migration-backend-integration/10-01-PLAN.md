  <behavior>
    SettingsStore:
    - get_defaults() returns dict of all use-case defaults from Redis hash
    - set_default(use_case, provider, model) stores use-case default as JSON in Redis hash
    - get_default(use_case) returns {'provider': str, 'model': str} or None
    - Uses key prefix 'aura:model_router:settings'

    KeyManager:
    - __init__ reads AURA_MASTER_KEY env var, creates Fernet cipher
    - __init__ without AURA_MASTER_KEY raises ValueError with clear message
    - store_key(provider, api_key) encrypts and stores, returns masked key
    - get_key(provider) decrypts and returns plaintext key or None
    - get_masked_key(provider) returns masked display string or None
    - delete_key(provider) removes key, returns bool
    - validate_key(provider, health_check_fn) validates by calling provider health_check
    - Masking: keys < 8 chars → '***', longer → first 3 + '...' + last 3
    - Uses key prefix 'aura:model_router:api_keys'

    ModelCache:
    - get_models(provider, router, redis_client, ttl_seconds=900) checks Redis cache first
    - On cache hit: deserializes and returns list[ModelInfo]
    - On cache miss: calls router.list_models(provider), stores in Redis with TTL, returns result
    - get_models(..., force_refresh=True) bypasses cache
    - Uses key pattern 'aura:models:{provider}'
  </behavior>
  <action>
**settings_store.py:** Follow Pattern 2 from 10-RESEARCH.md exactly. `SettingsStore` class with `__init__(self, redis_client)` — accepts an async Redis client (do NOT import redis). Key `SETTINGS_KEY = 'aura:model_router:settings'`. Methods: `async def get_defaults()` reads all fields via `hgetall`, JSON-deserializes values. `async def set_default(use_case, provider, model)` writes JSON `{'provider': provider, 'model': model}` via `hset`. `async def get_default(use_case)` reads single field via `hget`, returns parsed dict or None.

**key_manager.py:** Follow Pattern 3 from 10-RESEARCH.md. `KeyManager` class with `__init__(self, redis_client)` — reads `AURA_MASTER_KEY` from env. If missing, raise `ValueError('AURA_MASTER_KEY environment variable is required for API key encryption')`. Create `Fernet` instance from the key. `KEYS_HASH = 'aura:model_router:api_keys'`. Implement `_mask_key(key)`, `async store_key(provider, api_key)`, `async get_key(provider)`, `async get_masked_key(provider)`, `async delete_key(provider)`, `async validate_key(provider, health_check_fn)` where health_check_fn is an async callable that validates the key externally. Import `from cryptography.fernet import Fernet` at module level (cryptography is already a transitive dep of google-auth).

**cache.py:** Implement as a module-level async function `get_cached_models(provider, redis_client, router, ttl_seconds=900, force_refresh=False)`. Cache key pattern: `f'aura:models:{provider}'`. On cache hit (and not force_refresh): deserialize JSON list and return `[ModelInfo(**m) for m in data]`. On cache miss or force_refresh: call `await router.list_models(provider)`, serialize via `[m.model_dump() for m in models]`, store with `redis_client.setex(cache_key, ttl_seconds, json_data)`, return models. Also add a `ModelCache` class that wraps this for dependency injection: `__init__(self, redis_client, router, default_ttl=900)` with `async def get_models(provider, force_refresh=False)`.

**__init__.py:** Add exports for `SettingsStore`, `KeyManager`, `ModelCache`, `get_cached_models`. Add to `__all__`. Import lazily or directly — since these modules have no heavy dependencies (cryptography is small), direct import is fine.

Ensure all three modules:
- Accept Redis client via constructor (no internal `import redis`)
- Use `json.dumps` / `json.loads` for serialization (not pickle)
- Have module-level docstrings following Google Python style
- Use type annotations throughout
