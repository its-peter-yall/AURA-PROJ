ExpandedQuery model:
```python
class ExpansionTerm(BaseModel):
    term: str
    source_entity: str
    relationship: str
    weight: float

class ExpandedQuery(BaseModel):
    original_query: str
    expanded_query: str
    expansion_terms: List[ExpansionTerm]
    entities_found: List[str]
```

Entity identification approach:
1. Use embedding similarity to find entities matching query terms
2. Search entity names and definitions for query term matches
3. Combine results, deduplicate

Cypher for entity lookup:
```cypher
// Find entities by name similarity
MATCH (e:Entity)
WHERE e.module_id IN $module_ids OR $module_ids IS NULL
AND (toLower(e.name) CONTAINS toLower($query_term)
     OR toLower(e.definition) CONTAINS toLower($query_term))
RETURN e.id, e.name, e.type, e.definition
LIMIT 5
```

OR use vector search on entity embeddings:
```cypher
CALL db.index.vector.queryNodes('concept_vector_index', 5, $query_embedding)
YIELD node, score
WHERE score > 0.7
RETURN node.id, node.name, node.type, node.definition, score
```

Expansion term selection:
1. For each identified entity, get related entities via DEFINES, USES, RELATED_TO
2. Weight expansion terms by relationship confidence
3. Filter to top N terms by weight
4. Format expanded query: "original_query related_term1 related_term2 ..."

Integration with search:
1. Call expand_query() before search
2. Use expanded query for fulltext search
3. Use original query for vector search (embeddings capture semantics)
4. Return expansion info in results for transparency

```python
class QueryIntent(str, Enum):
    FACTUAL = "factual"           # "What is X?"
    CONCEPTUAL = "conceptual"     # "Explain Y"
    COMPARATIVE = "comparative"   # "Compare X and Y"
    PROCEDURAL = "procedural"     # "How to Z?"
    EXPLORATORY = "exploratory"   # "Tell me about X"

class QueryConstraint(BaseModel):
    type: str  # "topic", "time", "scope"
    value: str

class QueryAnalysis(BaseModel):
    original_query: str
    intent: QueryIntent
    key_terms: List[str]
    constraints: List[QueryConstraint]
    suggested_expansion: bool
```

Intent detection can use:
1. Simple keyword patterns ("what is", "how to", "compare", etc.)
2. Or LLM-based classification for complex queries

This service supports both query expansion and future features like answer synthesis.
