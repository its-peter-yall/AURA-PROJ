Tests for `_map_openrouter_error` should NOT import `openai` SDK — instead create mock exception classes that mimic the openai exception hierarchy names so tests work without the openai package installed:

```python
# Mock openai exceptions for error mapping tests (avoids requiring openai SDK)
class FakeOpenAIError(Exception):
    def __init__(self, message, response=None, body=None):
        super().__init__(message)
        self.response = response
        self.body = body
```

Create fake classes named to match what `_map_openrouter_error` checks via `isinstance()`. Since the error mapper does `import openai as openai_mod` and checks `isinstance(error, openai_mod.AuthenticationError)`, the tests must either:
1. Mock the openai module with fake classes, OR
2. Test the fallback path where openai import fails (returns generic ModelRouterError)

Best approach: Test error mapping by mocking `openai` as a module using `unittest.mock.patch.dict('sys.modules', ...)` with a fake module that has the exception classes. This keeps tests independent of the openai package.

Key implementation points:
- `_is_test_mode()` — same pattern as vertex_ai.py

- `health_check()` — test mode returns True, real mode checks /api/v1/auth/key via httpx
