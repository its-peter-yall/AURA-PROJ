# Plan 09-02: General Compute Key Validation — Both Settings Routers

Adds General Compute API key validation to both AURA-CHAT and AURA-NOTES-MANAGER backend settings routers, following the existing OpenRouter validation pattern.

## must_haves

- Both `AURA-CHAT/server/routers/settings.py` and `AURA-NOTES-MANAGER/api/routers/settings.py` import `GeneralComputeConfig` and `GeneralComputeProvider`
- Both have `GENERAL_COMPUTE_VALIDATION_METHOD = "health_check"` constant
- Both have `_validate_generalcompute_key(api_key)` function that creates a temp `GeneralComputeProvider` and calls `health_check()`
- Both `_validate_provider_key()` functions dispatch to `_validate_generalcompute_key` when `provider == ProviderType.GENERAL_COMPUTE.value`
- Both `_get_validation_method()` functions return `GENERAL_COMPUTE_VALIDATION_METHOD` for General Compute
- Existing OpenRouter and Vertex AI validation continues to work unchanged
