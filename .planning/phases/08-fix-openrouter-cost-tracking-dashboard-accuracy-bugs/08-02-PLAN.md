# Plan 08-02: Wire Pricing Cache Population into App Startup

## Goal

Call `CostCalculator.populate_openrouter_pricing()` during server startup in both
AURA-CHAT and AURA-NOTES-MANAGER so that the pricing cache is populated before the
first OpenRouter request is processed. This ensures all new usage records have
accurate cost estimates.

## must_haves

- AURA-CHAT `lifespan()` calls `populate_openrouter_pricing()` after
  `CostCalculator` is instantiated
- AURA-NOTES-MANAGER `_wire_usage_tracking()` calls `populate_openrouter_pricing()`
  after `CostCalculator` is instantiated
- Both apps handle pricing fetch failure gracefully (log warning, continue startup)
- Both apps only attempt pricing fetch when OpenRouter config has an API key
- Server startup is not blocked if OpenRouter pricing fetch fails or times out
