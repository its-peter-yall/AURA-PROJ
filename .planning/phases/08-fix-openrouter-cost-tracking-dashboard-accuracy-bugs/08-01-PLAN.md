# Plan 08-01: Shared Package — Pricing Cache + Streaming Usage

## Goal

Fix the two root causes in the shared `model_router` package that prevent accurate
OpenRouter cost tracking: (1) the empty pricing cache that makes all costs $0, and
(2) the `len(text) // 4` character-count token estimation used instead of real
streaming usage data from OpenRouter.

## must_haves

- `CostCalculator` has a method that fetches pricing from the OpenRouter `/api/v1/models`
  endpoint and populates `_openrouter_pricing` with per-1M-token values
- `router.py` `stream()` and `stream_with_usage()` capture the `usage` object from
  the final OpenRouter SSE chunk instead of falling back to character-count estimation
- `StreamChunk` can carry optional usage data so the final chunk can propagate real
  token counts from the provider
- Character-count estimation remains as fallback for interrupted streams and non-OpenRouter
  providers
- All existing tests continue to pass; new tests cover pricing fetch and streaming usage
