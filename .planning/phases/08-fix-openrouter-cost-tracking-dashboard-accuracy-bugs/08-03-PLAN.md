# Plan 08-03: Fix Date Range Filters — End-of-Day for Same-Day Queries

## Goal

Fix the `_parse_date_range` function in both apps' usage routers so that date-only
`end_date` query parameters include the entire day's data instead of truncating at
midnight (00:00:00).

## must_haves

- `_parse_date_range` in both `AURA-CHAT/server/routers/usage.py` and
  `AURA-NOTES-MANAGER/api/routers/usage.py` adjusts `end_date` to 23:59:59.999999
  when the time component is midnight
- Date-only queries like `?end_date=2026-05-16` include all records from that day
- Explicit time-component queries (e.g., `?end_date=2026-05-16T14:30:00`) are
  unchanged
- Unit tests verify both same-day and explicit-time behaviors

## Tasks

<task id="08-03-T1" title="Fix _parse_date_range in AURA-CHAT usage router">
  <read_first>
    - AURA-CHAT/server/routers/usage.py (lines 54-78 — _parse_date_range function)
  </read_first>
  <action>
    Modify `_parse_date_range` in `AURA-CHAT/server/routers/usage.py`:

    After the existing `end_date` timezone handling (line 71:
    `end_date = end_date.replace(tzinfo=timezone.utc)`), add a check:

    If `end_date` was provided (not None) AND its time component is midnight
    (hour=0, minute=0, second=0, microsecond=0), replace the time component
    with end-of-day: `end_date = end_date.replace(hour=23, minute=59, second=59, microsecond=999999)`

    This goes AFTER the tzinfo assignment but BEFORE the return statement.

    The check should be:
    ```
    if (end_date.hour == 0 and end_date.minute == 0
            and end_date.second == 0 and end_date.microsecond == 0):
        end_date = end_date.replace(
            hour=23, minute=59, second=59, microsecond=999999,
        )
    ```

    This only fires when the caller passes a date-only string like
    `?end_date=2026-05-16` which FastAPI parses as midnight. If an
    explicit time is provided, the adjustment is skipped.

    Important: This adjustment must only happen when `end_date` was explicitly
    provided by the caller (not when it defaults to `now`). The check should
    be placed inside the `elif` branch (i.e., after verifying end_date was
    not None).
  </action>
  <acceptance_criteria>
    - `AURA-CHAT/server/routers/usage.py` `_parse_date_range` adjusts midnight end_date to 23:59:59.999999
    - Adjustment only fires when end_date time is exactly 00:00:00.000000
    - When end_date is None (defaults to now), no adjustment occurs
    - When end_date has explicit non-midnight time, no adjustment occurs
    - `python -m pytest AURA-CHAT/tests/ -k usage -x` exits 0
  </acceptance_criteria>
</task>
