```python
def _validate_label(
    label: str,
    allowlist: set,
    label_kind: str = "label",
) -> str:
    """Validate a dynamic Cypher label/type against an allowlist.

    Args:
        label: The label or relationship type string.
        allowlist: Set of allowed values.
        label_kind: Human-readable kind for error messages.

    Returns:
        The validated label string.

    Raises:
        ValueError: If label is empty or not in the allowlist.
    """
    if not isinstance(label, str) or not label.strip():
        raise ValueError(f"Invalid empty {label_kind}")
    cleaned = label.strip()
    if cleaned not in allowlist:
        raise ValueError(
            f"Disallowed {label_kind} '{cleaned}'. "
            f"Allowed: {sorted(allowlist)}"
        )
    return cleaned
```

### T5: Firestore Server-Side Pagination

**Decision ref:** D-05
**File:** `AURA-NOTES-MANAGER/api/modules/service.py` (lines 162–169)

**Problem:** `list(query.stream())` loads the entire collection into memory before slicing. For collections with thousands of documents, this wastes memory and Firestore read units.

**Change:**

```diff
-        # Get total count (note: Firestore doesn't have efficient count)
-        all_docs = list(query.stream())
-        total = len(all_docs)
-
-        # Apply pagination
-        offset = (page - 1) * page_size
-        paginated_docs = all_docs[offset : offset + page_size]
+        # Server-side pagination using Firestore offset/limit
+        # (avoids loading entire collection into memory)
+        try:
+            count_query = query.count()
+            count_result = count_query.get()
+            total = count_result[0][0].value
+        except Exception:
+            # Fallback: unknown total if count() unsupported
+            total = -1
+
+        offset = (page - 1) * page_size
+        paginated_docs = list(
+            query.offset(offset).limit(page_size).stream()
+        )
```

> **Note:** The `query.count()` aggregation requires `google-cloud-firestore >= 2.11.0`. If unavailable, `-1` signals "unknown total" — the frontend should handle this gracefully (e.g., disable page count display).

**Verify:**
- [ ] `list(query.stream())` no longer loads full collection
- [ ] `offset()` and `limit()` are applied at the Firestore query level
- [ ] `total` is either accurate (via `count()`) or `-1` (fallback)
- [ ] API response contract unchanged (`modules`, `total`, `page`, `page_size`)
- [ ] Existing module list tests pass
