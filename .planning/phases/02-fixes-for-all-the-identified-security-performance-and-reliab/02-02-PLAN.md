## Overview

This plan implements `batch_create_relationships()` in `GraphManager` to replace the sequential `create_relationship()` loop in `document_processor.py`. The batch method uses Neo4j's `UNWIND` pattern to reduce network round-trips from ~50 per document to ~3-5 (one per relationship type group).

**Dependency:** This plan requires the `_validate_label()` helper and `ALLOWED_NODE_LABELS` / `ALLOWED_REL_TYPES` constants from Plan 01 (T3).

#### graph_manager.py — Add batch method after `create_relationship` (after line ~421)

```python
async def batch_create_relationships(
    self,
    relationships: list[dict[str, Any]],
) -> int:
    """Create multiple relationships in batched UNWIND queries.

    Groups relationships by (source_type, rel_type, target_type)
    and executes one UNWIND query per group. All labels are
    validated against the allowlist before execution.

    Args:
        relationships: List of dicts with keys:
            source_type, source_id, rel_type,
            target_type, target_id, properties.

    Returns:
        Total number of relationships created/merged.

    Raises:
        ValueError: If any label/type fails allowlist
            validation.
    """
    if not relationships:
        return 0

    if self._test_mode:
        self._relationships.extend(relationships)
        return len(relationships)

    # Group by (source_type, rel_type, target_type) so each
    # UNWIND query uses a single Cypher label pattern.
    from collections import defaultdict
    groups: dict[
        tuple[str, str, str], list[dict[str, Any]]
    ] = defaultdict(list)

    for rel in relationships:
        key = (
            rel["source_type"],
            rel["rel_type"],
            rel["target_type"],
        )
        groups[key].append(
            {
                "source_id": rel["source_id"],
                "target_id": rel["target_id"],
                "properties": rel.get("properties") or {},
            }
        )

    total_created = 0
    for (src_type, rel_type, tgt_type), batch in (
        groups.items()
    ):
        # Validate all labels against allowlists
        safe_src = _validate_label(
            src_type,
            config.ALLOWED_NODE_LABELS,
            "node label",
        )
        safe_rel = _validate_label(
            rel_type,
            config.ALLOWED_REL_TYPES,
            "relationship type",
        )
        safe_tgt = _validate_label(
            tgt_type,
            config.ALLOWED_NODE_LABELS,
            "node label",
        )

        query = f"""
        UNWIND $batch AS item
        MATCH (a:{safe_src} {{id: item.source_id}})
        MATCH (b:{safe_tgt} {{id: item.target_id}})
        MERGE (a)-[r:{safe_rel}]->(b)
        SET r += item.properties
        RETURN count(r) AS created
        """
        result = await self.execute_query(
            query, {"batch": batch}
        )
        if result and result[0].get("created"):
            total_created += result[0]["created"]

    logger.info(
        "Batch created %d relationships in %d groups",
        total_created,
        len(groups),
    )
    return total_created
```

## Performance Expectations

| Metric | Before | After |
|--------|--------|-------|
| Neo4j round-trips per document | ~50 (1 per relationship) | ~3-5 (1 per type group) |
| Network overhead | High (50× connection setup) | Low (3-5× connection setup) |
| Transaction semantics | Individual MERGE per rel | Batched MERGE via UNWIND |
| Memory (Neo4j server) | Low per query | Moderate (batch in memory) |

**Batch size safety:** Typical documents have ~50 relationships. UNWIND with 50 items is well within Neo4j's safe batch range (recommended: up to 500-1000). No chunking needed for this use case.

---

## Risk Mitigations

| Risk | Mitigation |
|------|-----------|
| UNWIND changes MERGE semantics | MERGE is idempotent — same behavior whether called 50× individually or 1× with UNWIND |
| Large batch causes OOM on Neo4j | Typical document has ~50 rels; well below the 500-1000 recommendation |
| Mixed types in one UNWIND | Groups by type triplet — each UNWIND has uniform label patterns |
| Partial batch failure | Neo4j transactions are atomic — entire UNWIND succeeds or fails, same as individual calls |
