**compat.py:** Create `VertexCompatModel` class and `VertexCompatResponse` helper.

```python
class VertexCompatResponse:
    """Response object matching GenerativeModel.generate_content() return shape."""
    def __init__(self, text: str):
        self.text = text
        # Build candidates structure for callers that access response.candidates[0].content.parts[0].text
        self.candidates = [_CompatCandidate(text)]

class _CompatCandidate:
    def __init__(self, text: str):
        self.content = _CompatContent(text)

class _CompatContent:
    def __init__(self, text: str):
        self.parts = [_CompatPart(text)]

class _CompatPart:
    def __init__(self, text: str):
        self.text = text
```

`VertexCompatModel.__init__(self, model_name: str)`: Store model name. Lazy router reference.

`generate_content(self, contents, generation_config=None, safety_settings=None)`:
- Import `get_default_router` lazily.
- Extract config fields from `generation_config` (handle both dict and GenerationConfig objects — use `getattr` for temperature, max_output_tokens, response_mime_type just like AURA-CHAT's `_GenerativeModelWrapper`).
- Build kwargs dict: `model=self._model_name, contents=contents, temperature=..., max_output_tokens=...`.
- Call `router.generate(**kwargs)` — but this is async. Use `asyncio.run()` if no event loop running, otherwise `loop.run_until_complete()`. Wrap in try/except for RuntimeError (nested event loop) and fall back to creating a new loop.
- Convert `GenerateResponse.text` → `VertexCompatResponse(text)`.
- On any `ModelRouterError`, re-raise as the app-appropriate error (import `VertexAIRequestError` from calling context is tricky — so just re-raise the ModelRouterError, which is already an Exception subclass and will be caught by existing `except Exception` handlers).

Also create a `_run_sync(coro)` helper that handles the async-to-sync bridge cleanly:
```python
def _run_sync(coro):
    try:
        loop = asyncio.get_running_loop()
    except RuntimeError:
        return asyncio.run(coro)
    # If there's already a running loop (e.g., inside FastAPI), use run_in_executor or thread
    import concurrent.futures
    with concurrent.futures.ThreadPoolExecutor(max_workers=1) as pool:
        return pool.submit(asyncio.run, coro).result()
```
