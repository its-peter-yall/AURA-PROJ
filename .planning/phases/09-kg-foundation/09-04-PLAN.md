<requirements>
1. Extract 9 relationship types:
   - DEFINES: A defines/explains B
   - DEPENDS_ON: A requires/needs B
   - USES: A uses/applies B
   - SUPPORTS: A provides evidence for B
   - CONTRADICTS: A conflicts with B
   - EXTENDS: A builds upon B
   - IMPLEMENTS: A is an implementation of B
   - REFERENCES: A cites/mentions B
   - RELATED_TO: Generic relationship

2. Each relationship has: source_entity, target_entity, relationship_type, confidence
3. Two-pass extraction: entities first, then relationships between them
4. Relationship prompt includes previously extracted entities for context
5. Batch relationship extraction to minimize API calls
6. Store relationships as Neo4j edges with properties
</requirements>

2. `_build_relationship_prompt(self, text: str, entities: List[Entity]) -> str`
3. `extract_entities_and_relationships(self, text: str) -> Tuple[List[Entity], List[Relationship]]`

Add Relationship Pydantic model:
```python
class Relationship(BaseModel):
    source_entity: str  # Entity name
    target_entity: str  # Entity name
    relationship_type: Literal["DEFINES", "DEPENDS_ON", "USES", "SUPPORTS", "CONTRADICTS", "EXTENDS", "IMPLEMENTS", "REFERENCES", "RELATED_TO"]
    confidence: float = Field(ge=0.0, le=1.0)
    evidence: Optional[str] = None  # Text supporting the relationship
```

Relationship prompt design:
- Include list of extracted entities with their definitions
- Ask LLM to identify relationships between them
- Request supporting evidence from the text
- Handle case where no relationships exist

The two-pass approach:
1. First call: extract_entities() returns entities
2. Second call: extract_relationships() with entities as context
3. Combined call: extract_entities_and_relationships() does both

Validation:
- Ensure source_entity and target_entity exist in extracted entities
- Filter out self-relationships (A -> A)
- Deduplicate identical relationships

<action>
Integrate relationship extraction into the KG processing pipeline.

Modifications:
1. After entity extraction, call extract_relationships()
2. Store relationships in Neo4j as edges between entity nodes
3. Add relationship properties: confidence, evidence, created_at

New method: `_store_entity_relationships(self, relationships: List[Relationship], module_id: str)`

Cypher pattern for relationship storage:
```cypher
MATCH (source:Entity {name: $source_name, module_id: $module_id})
MATCH (target:Entity {name: $target_name, module_id: $module_id})
MERGE (source)-[r:$rel_type]->(target)
SET r.confidence = $confidence,
    r.evidence = $evidence,
    r.created_at = datetime()
```

Note: Neo4j doesn't support parameterized relationship types, so use conditional MERGE or APOC.

Handle edge cases:
- Relationship references entity not yet stored (skip or queue)
- Multiple relationships between same entity pair (keep highest confidence)
- Cross-chunk relationships (entities from different chunks)
</action>
