Class: `EntityDeduplicator`
- `__init__(self, similarity_threshold: float = 0.85)`
- `deduplicate(self, entities: List[Entity], embeddings: Dict[str, List[float]]) -> Tuple[List[Entity], Dict[str, str]]`
- `_compute_similarity(self, emb1: List[float], emb2: List[float]) -> float`
- `_find_duplicates(self, entities: List[Entity], embeddings: Dict[str, List[float]]) -> List[Tuple[Entity, Entity]]`
- `_merge_entities(self, entity1: Entity, entity2: Entity) -> Entity`

Algorithm:
1. Build similarity matrix for all entity pairs
2. Identify pairs above threshold (0.85)
3. Use Union-Find to group all duplicates transitively
4. For each group, select canonical entity (highest confidence)
5. Return deduplicated list and mapping (old_name -> canonical_name)

Merge strategy:
- Keep entity with higher confidence_score as canonical
- Combine definitions if significantly different
- Aggregate mention counts
- Track merge history for debugging

Cosine similarity implementation:
```python
def _compute_similarity(self, emb1: List[float], emb2: List[float]) -> float:
    dot_product = sum(a * b for a, b in zip(emb1, emb2))
    norm1 = math.sqrt(sum(a * a for a in emb1))
    norm2 = math.sqrt(sum(b * b for b in emb2))
    return dot_product / (norm1 * norm2) if norm1 and norm2 else 0.0
```

Consider using numpy for efficiency if many entities.

Configuration:
```python
ENTITY_DEDUP_SIMILARITY_THRESHOLD = 0.85
```

Cypher for merging entities:
```cypher
// Update relationships from old entity to canonical
MATCH (c:Chunk)-[r:CONTAINS_ENTITY]->(old:Entity {name: $old_name, module_id: $module_id})
MATCH (canonical:Entity {name: $canonical_name, module_id: $module_id})
MERGE (c)-[:CONTAINS_ENTITY]->(canonical)
SET canonical.mention_count = canonical.mention_count + old.mention_count
DELETE r
// Optionally delete old entity or mark as alias
DETACH DELETE old
```

Handle edge cases:
- Entity relationships need to point to canonical
- Circular references (A->B, B->C, A->C)
- Previously deduplicated entities (idempotency)
