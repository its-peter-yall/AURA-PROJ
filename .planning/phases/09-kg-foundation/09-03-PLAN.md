# 09-03-PLAN: LLM Entity Extractor

<objective>
Port AURA-CHAT's LLM-based entity extraction to AURA-NOTES-MANAGER.

Purpose: Extract structured entities (Topic, Concept, Methodology, Finding) from document chunks using Gemini with configurable prompts and confidence scoring.

Output: New `services/llm_entity_extractor.py` module with structured extraction capabilities.
</objective>

<requirements>
1. Extract 4 entity types: Topic, Concept, Methodology, Finding
2. Each entity has: name, type, definition, confidence_score (0.0-1.0)
3. Use Gemini (gemini-1.5-flash) for extraction
4. Structured JSON output with validation
5. Batch processing (LLM_ENTITY_BATCH_SIZE = 3000 tokens)
6. Retry logic with exponential backoff
7. Rate limiting to respect API quotas
8. Entity schema matching AURA-CHAT's structure
</requirements>

<tasks>
<task>
<type>create</type>
<file>AURA-NOTES-MANAGER/services/llm_entity_extractor.py</file>
<action>
Create the LLM entity extractor service.

Purpose: Extract structured entities from text using Gemini with validated JSON output.

The module should include:

1. `LLMEntityExtractor` class with:
   - `__init__(self, model_name: str = "gemini-1.5-flash")`
   - `extract_entities(self, text: str, entity_types: List[str] = None) -> List[Entity]`
   - `_build_extraction_prompt(self, text: str, entity_types: List[str]) -> str`
   - `_parse_response(self, response: str) -> List[Entity]`

2. Extraction prompt template (port from AURA-CHAT):
   - Clearly specify output JSON schema
   - Include examples for each entity type
   - Request confidence scores
   - Handle edge cases (no entities found, ambiguous entities)

3. Entity Pydantic model:
   ```python
   class Entity(BaseModel):
       name: str
       type: Literal["Topic", "Concept", "Methodology", "Finding"]
       definition: str
       confidence_score: float = Field(ge=0.0, le=1.0)
       source_text: Optional[str] = None
   ```

4. Configuration constants:
   - LLM_ENTITY_BATCH_SIZE = 3000
   - MAX_RETRIES = 3
   - RETRY_BACKOFF_BASE = 2.0

5. Error handling:
   - GeminiAPIError for API failures
   - ValidationError for invalid JSON responses
   - Graceful degradation when extraction fails

Reference AURA-CHAT's llm_entity_extractor.py for prompt structure and error handling patterns.

Integration: Use existing Gemini client from AURA-NOTES-MANAGER if available, otherwise create new.
</action>

</verify>
