# 09-02-PLAN: Hierarchical Chunking

<objective>
Port AURA-CHAT's hierarchical parent-child chunking to AURA-NOTES-MANAGER.

Purpose: Enable richer context retrieval by creating ParentChunk nodes (1500 tokens) containing child Chunk nodes (800 tokens), improving RAG quality.

Output: kg_processor.py enhanced with hierarchical chunking, creating ParentChunk→Chunk structure.
</objective>

<requirements>
1. Parent chunks: 1500 tokens with minimal overlap
2. Child chunks: 800 tokens with 200 token overlap
3. Each parent chunk contains 1-3 child chunks
4. Both parent and child chunks get embeddings (768-dim via Gemini)
5. Create HAS_PARENT_CHUNK relationship from Document to ParentChunk
6. Create HAS_CHILD relationship from ParentChunk to Chunk
7. Maintain backward compatibility with existing chunking (feature flag)
8. Position tracking for both parent and child chunks
</requirements>

Enhance kg_processor.py with hierarchical chunking capability.

Add `chunk_text_hierarchical()` method that:
1. First splits text into parent chunks (1500 tokens)
2. Then splits each parent into child chunks (800 tokens, 200 overlap)
3. Generates embeddings for both parent and child chunks
4. Returns structured data for both levels

Add configuration constants at module level:
- PARENT_CHUNK_SIZE = 1500
- CHILD_CHUNK_SIZE = 800
- CHILD_CHUNK_OVERLAP = 200

Add feature flag `use_hierarchical_chunking` parameter to processing functions.

Reference AURA-CHAT's document_processor.py for the algorithm:
- Text normalization before chunking
- Sentence boundary awareness (avoid mid-sentence splits)
- Token counting using tiktoken or similar

Integration points:
- Modify `process_document()` to use hierarchical chunking when flag is True
- Store parent chunks with `_store_parent_chunks()` method
- Link child chunks to parents via HAS_CHILD relationship

Pitfalls to avoid:
- Don't break in middle of words or sentences
- Ensure child chunks sum to cover parent text
- Handle documents shorter than PARENT_CHUNK_SIZE gracefully
