Create React Query hook for per-document KG processing operations.

Purpose:
- Manage per-document status and processing
- Provide status polling for long-running tasks
- Handle batch processing mutations

Operations:
1. useFileKGStatus(document_id) - Fetch KG status for single document
   - Endpoint: GET /api/v1/kg/documents/{document_id}/status

2. useProcessFiles - Mutation to trigger batch processing (multiple files)
   - Endpoint: POST /api/v1/kg/process-batch
   - Accepts array of document IDs
   - Returns task_id for progress tracking

3. useProcessingQueue - Fetch documents currently processing
   - Endpoint: GET /api/v1/kg/processing-queue

4. useTaskProgress - Fetch Celery task progress
   - Endpoint: GET /api/v1/kg/tasks/{task_id}/status

"Process All in Module" workflow:
- Client-side: Query all documents in module with kg_status=pending
- Client-side: Extract document IDs
- Call useProcessFiles with pending document IDs
- Backend processes only pending docs (skips ready docs)

Key Feature: Batch processing accepts array of file IDs
- All files tagged with same module_id
- Progress tracked per document
- Errors tracked per document
- Idempotent (backend skips ready docs)

Integration:
- Call explorerApi endpoints
- Handle async processing with polling (refetchInterval for progress)
- Invalidate queries on completion
