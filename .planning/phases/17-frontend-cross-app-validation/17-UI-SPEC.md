## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#000000` (`bg-background`) | Page backgrounds, body surfaces |
| Secondary (30%) | `#0d0d0d` (`bg-card`) / `#1a1a1a` (`bg-secondary`) | Cards, sections, sidebar, nav |
| Accent (10%) | `#FFD400` (`text-primary`) | See reserved-for list below |
| Destructive | `#dc2626` (`text-destructive`) | Destructive actions only |

**Source:** Pre-populated from both `index.css` files — identical `@theme` tokens in AURA-CHAT and AURA-NOTES-MANAGER.

Accent reserved for (explicit list):
- Section heading icons (Shield, Cpu, Key, Zap icons next to section titles)
- Selected model item highlight (bg-primary/20, text-primary)
- Input focus ring (`ring-primary/30`)
- Status badges for healthy/connected/ready states (green-500 used here, not yellow)
- Mutations feedback — success check icon (green-500 used here, not yellow)
- Sidebar active item left border and text color

**Destructive reserved for:**
- Failed update feedback (AlertCircle icon + `text-destructive`)
- API key delete actions
- System status disconnected/not-ready badges

**Supporting semantic colors:**
- `#22c55e` (`text-green-500`): Success states, healthy status badges
- `#a6a6a6` (`text-muted-foreground`): Descriptions, placeholder text, timestamps
- `#262626` (`bg-muted`): Hover states, input backgrounds
- `#333333` (`border-border`): Card borders, dividers

## Interaction Contracts

### AURA-NOTES-MANAGER — DefaultModelSection Update

| Interaction | Behavior |
|-------------|----------|
| Page load | TanStack Query fetches `/v1/settings/defaults` and `/v1/settings/models`. Skeleton pulse shown during load (3 placeholder rows). |
| Use case picker open | Click trigger → AnimatePresence slide-down panel with search, provider grouping. Framer Motion `duration: 0.15`. |
| Model select | Click model → `onChange` fires → `useUpdateDefault` mutation → optimistic UI with `isPending` pulse text → success Check icon (green-500) or error AlertCircle (destructive). |
| Cross-tab sync | `useEffect` on `currentValue` syncs local state if server default changes from another tab. |
| Model type filtering | `USE_CASE_MODEL_TYPES` maps each use case to `'generation'` or `'embedding'`. Gatekeeper and relationship_extraction both use `'generation'` (only generation models shown in picker). |

### AURA-CHAT — Settings Page (new)

| Interaction | Behavior |
|-------------|----------|
| Navigation | Sidebar "Settings" link → `/settings` route → `RoleProtectedRoute` with `requiredRole="admin"`. |
| Page layout | 2/3 + 1/3 grid: Main config (left) + System Status sidebar (right). Mirror NOTES-MANAGER layout exactly. |
| Provider section | `ProviderSettingsSection` shows provider cards with model counts. Identical to NOTES-MANAGER. |
| Default Models section | 5 use case rows (chat, embeddings, entity_extraction, gatekeeper, relationship_extraction). Same `HierarchicalModelPicker` per row. |
| API Keys section | `ApiKeyManager` with store/delete/validate per provider. Identical to NOTES-MANAGER. |
| System Status | Health query with 30s refetch. Shows Chat Server, Neo4j, Redis, Semantic Router status. |

| Test | Interaction | Assertion |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Touch targets | `min-h-11` (44px) on mobile for all interactive elements |
| Focus indicators | `ring-1 ring-primary/30` on focused inputs and pickers |
| Color contrast | `#FFD400` on `#000000` = 13.8:1 (WCAG AAA) |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` disables all animations in `index.css` |
| Screen reader | Status badges have `aria-label` equivalent via visible text on desktop (`hidden sm:inline`) |
| Keyboard navigation | All pickers, buttons, and form inputs are focusable and operable via keyboard |
