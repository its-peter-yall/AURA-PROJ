<task>
<type>create</type>
<file>AURA-NOTES-MANAGER/services/embeddings.py</file>
<action>
Create embedding service for AURA-NOTES-MANAGER.

Purpose: Centralized embedding generation with batching, rate limiting, and retry logic.

Class: `EmbeddingService`
- `__init__(self, model_name: str = "gemini-embedding-001")`
- `embed_text(self, text: str) -> List[float]`
- `embed_batch(self, texts: List[str], batch_size: int = 100) -> List[List[float]]`
- `embed_entity(self, entity: Entity) -> List[float]`
- `embed_entities(self, entities: List[Entity]) -> Dict[str, List[float]]`

Key features:
1. Batch processing: group multiple texts into single API call
2. Rate limiting: configurable requests per minute (default: 60)
3. Retry logic: exponential backoff on API errors
4. Caching: optional in-memory cache for repeated texts
5. Model fallback: if primary model fails, try secondary

Configuration constants:
```python
EMBEDDING_MODEL = "gemini-embedding-001"
EMBEDDING_DIMENSIONS = 768
EMBEDDING_BATCH_SIZE = 100
RATE_LIMIT_RPM = 60
MAX_RETRIES = 3
```

Entity embedding format: "{entity.name}: {entity.definition}"
- Combines name and definition for richer representation
- Truncate if exceeds model's token limit
- Handle entities without definitions (use name only)

Reference AURA-CHAT's embeddings.py for patterns, but adapt to AURA-NOTES-MANAGER's structure.
</action>

Process:
1. Collect all entities needing embeddings
2. Batch generate embeddings via EmbeddingService
3. Update Neo4j entities with embedding property
4. Log statistics (entities embedded, avg time per entity)

Cypher for updating entity embeddings:
```cypher
MATCH (e:Entity {id: $entity_id})
SET e.embedding = $embedding,
    e.embedded_at = datetime()
```

Handle edge cases:
- Entity already has embedding (skip or re-embed based on flag)
- Embedding generation fails (log warning, continue with remaining)
- Empty definition (embed name only)
