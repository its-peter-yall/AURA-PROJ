**New implementation pattern for both files:**

```python
async def _validate_provider_key(
    provider: str,
    key_manager: KeyManager,
) -> bool | None:
    """Validate a stored provider key.

    Returns True if valid, False if invalid, None if validation
    is not applicable for this provider type.
    """
    validator: Callable[[str], Awaitable[bool]] | None = None

    if provider == ProviderType.OPENROUTER.value:
        validator = _validate_openrouter_key
    elif provider == ProviderType.VERTEX_AI.value:
        # Vertex AI uses service account credentials, not stored API keys.
        # Validate by checking if the router's Vertex provider can initialize.
        try:
            router = get_default_router()
            vertex_provider = router.get_provider(ProviderType.VERTEX_AI)
            return await vertex_provider.health_check()
        except Exception:
            return False
    elif provider == ProviderType.OLLAMA.value:
        # Ollama is local — no API key needed. Return None to indicate
        # validation is not applicable.
        return None

    if validator is None:
        return None

    return await key_manager.validate_key(provider, validator)
```

**Update the store_api_key and validate_api_key endpoints** in both files to handle the `None` return:

In `store_api_key()` and `validate_api_key()`, change the response to include a `validation_method` field:
```python
is_valid = await _validate_provider_key(provider, key_manager)
return {
    "provider": provider,
    "masked_key": masked_key,
    "valid": is_valid,  # True, False, or None
}
```
The `None` value serializes to JSON `null`, which tells the frontend "validation not applicable for this provider." This is more informative than always returning `False`.

**Both files must be updated identically** to maintain the cross-app parity established in Plan 10-02.
